## Что за проект

**Буквик** — детское веб-приложение для изучения русского алфавита.
Персонаж — львёнок Барри — озвучивает буквы, слова и задания.
Целевая аудитория: дети 3–6 лет, которые ещё не умеют читать.
Управление голосом: ребёнок не читает задания, Барри всё говорит вслух.

Стек: чистый HTML + CSS + JS (без фреймворков), 3D-персонаж через model-viewer.

## Как работает озвучка

1. Приложение пытается воспроизвести MP3 из `assets/audio/`
2. Если файл отсутствует — fallback на браузерный Web Speech API (`SpeechSynthesisUtterance`)
3. Скорость речи регулируется кнопками (0.6 / 0.78 / 0.92)
4. Тексты для генерации хранятся в `voice-lines.json`

## Задача: подключить TTS API для озвучки

**Проблема:** автор вручную записывал каждую букву — это долго.
**Решение:** использовать TTS API, запустить скрипт один раз — все файлы готовы.

### Вариант A — ElevenLabs (приоритет)

- Лучшее качество, живые русские голоса
- Бесплатный план: 10 000 символов/месяц (хватит для всего приложения)
- Сайт: https://elevenlabs.io

```bash
ELEVENLABS_API_KEY="ключ" \
ELEVENLABS_VOICE_ID="id_голоса" \
python3 generate_elevenlabs_audio.py
```

Флаг `--force` — перезаписать уже существующие файлы.

Рекомендуемые русские голоса: поиск по языку "Russian" в библиотеке голосов ElevenLabs.
Модель по умолчанию: `eleven_multilingual_v2`.

### Вариант B — Azure TTS

- Бесплатный план: 500 000 символов/месяц (Neural F0)
- Голос: `ru-RU-SvetlanaNeural` — очень натуральный

```bash
AZURE_SPEECH_KEY="ключ" \
AZURE_SPEECH_REGION="регион" \
python3 generate_azure_audio.py
```

## Правила работы (договорённости)

- Не трогать существующие MP3-файлы в `letters/` без явного запроса — они записаны вручную
- Не добавлять фреймворки (React, Vue и т.д.) — проект намеренно без зависимостей
- Не добавлять комментарии и документацию в код без запроса
- Не усложнять: если задачу можно решить просто — решать просто
- Все изменения в логике речи проверять через fallback (браузерный TTS работает без файлов)
